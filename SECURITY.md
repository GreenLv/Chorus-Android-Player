# Security policy

Please report security or privacy issues privately through the GitHub repository's security-advisory form. Do not open a public issue containing an exploit, credential, private endpoint, library, certificate, packet capture, or full log.

Before sharing diagnostics, remove schemes/hosts/ports, URL paths and queries, device identifiers, tokens, local paths, and server addresses. This repository does not accept private native providers for analysis or redistribution.

Only the latest source on the default branch is maintained during the initial research release. No APK or prebuilt native library is an official release artifact.
